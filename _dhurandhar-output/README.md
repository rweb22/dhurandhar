# Dhurandhar Output Directory

This directory contains all artifacts generated by Dhurandhar skills during the development process.

## Directory Structure

```
_dhurandhar-output/
├── phase-1-ideation/
│   ├── core-idea.md              # One-sentence vision
│   ├── brainstorming-report.md   # Alternatives and edge cases
│   └── product-brief.md          # 2-page strategic vision
│
├── phase-2-requirements/
│   └── prd.md                    # Product Requirements Document
│
├── phase-3-system-design/
│   ├── core-entities.md          # Data model
│   ├── api-design.md             # API contracts
│   ├── hld.md                    # High-level design
│   ├── lld.md                    # Low-level design
│   └── schemas/                  # Database schemas
│       └── schema.sql
│
├── phase-4-implementation-planning/
│   ├── epics-and-stories.md      # Implementation roadmap
│   ├── e2e-tests.md              # Test documentation
│   └── test-suites/              # Executable tests
│       ├── api-contracts/
│       ├── user-flows/
│       ├── error-cases/
│       └── fixtures/
│
└── phase-5-implementation/
    └── [Your actual source code]
```

## Workflow

Each phase builds on the previous:

1. **Phase 1 (Yudhishthira)**: Define the vision
2. **Phase 2 (Sahadeva)**: Define requirements with foresight
3. **Phase 3 (Arjuna)**: Design the system with precision
4. **Phase 4 (Nakula)**: Plan implementation and write tests
5. **Phase 5 (Bhima)**: Build and deploy

## Git Strategy

This directory should be committed to version control. It contains:
- ✅ Design documents (valuable for onboarding, decision history)
- ✅ Test specifications (define system behavior)
- ✅ Requirements (source of truth)

Add to `.gitignore` if you prefer to keep it local only:
```
_dhurandhar-output/
```

## Usage

Dhurandhar skills automatically read from and write to this directory:
- Skills read previous phase outputs for context
- Skills write their artifacts to the appropriate phase directory
- All artifacts follow consistent naming conventions

## Example Flow

```bash
# Phase 1: Ideation
/yudhishthira
/core-idea           # Creates: phase-1-ideation/core-idea.md
/brainstorming       # Creates: phase-1-ideation/brainstorming-report.md
/product-brief       # Creates: phase-1-ideation/product-brief.md

# Phase 2: Requirements
/sahadeva
/prd                 # Creates: phase-2-requirements/prd.md

# Phase 3: Design
/arjuna
/core-entities       # Creates: phase-3-system-design/core-entities.md
/api-design          # Creates: phase-3-system-design/api-design.md
/hld                 # Creates: phase-3-system-design/hld.md
/lld                 # Creates: phase-3-system-design/lld.md

# Phase 4: Planning
/nakula
/epics-and-stories   # Creates: phase-4-implementation-planning/epics-and-stories.md
/e2e-api-tests       # Creates: phase-4-implementation-planning/e2e-tests.md

# Phase 5: Build
/bhima
/implement           # Writes code, makes tests pass
/deploy              # Ships to production
```
